Compiler, Interpreuter, Linker

• "translator" biasanya merujuk kepada program atau perangkat lunak yang mengubah kode sumber dari satu bahasa pemrograman ke bahasa pemrograman lain.
• Ada 2 jenis translator utama pada bahasa pemrograman, yaitu Compiler dan Interpreter.

Compiler

• Compiler adalah jenis translator yang menerjemahkan seluruh kode sumber dalam satu kali proses ke dalam bahasa mesin atau kode antara (intermediate code).
• Compiler menghasilkan file biner atau kode antara yang dapat dieksekusi secara langsung oleh komputer.
• Contoh bahasa yang menggunakan compiler adalah C, C++, dan Ada.

Interpreter

• Interpreter adalah jenis translator yang membaca dan mengeksekusi kode sumber baris demi baris saat program berjalan.
• Contoh bahasa yang menggunakan interpreter adalah Python, JavaScript, dan Ruby.

Linker

• Tugas dari linker adalah menggabungkan berbagai library external dengan object file untuk menghasilkan sebuah program.
• kode program C biasanya butuh file library external. Sebagai contoh, untuk menampilkan teks ke layar dalam bahasa C perintahnya adalah printf(“Hello World”). Untuk bisa memproses perintah printf(), C butuh sebuah file external, yakni stdio.h
• Linker-lah yang akan menggabungkan hello_world.obj dengan stdio.h. Hasil akhirnya berupa sebuah program berextension *.exe. Karena file source code saya buat sebagai hello_world.c, file programnya akan bernama hello_world.exe. Dengan men- double-klik file hello_world.exe, program kita sudah langsung berjalan.

Perbedaan Compiler dan Interpreter

1. Compilermenyimpan Bahasa Mesin dalam bentuk Kode Mesin pada disk.
Interpreter tidak menyimpan Bahasa Mesin.
2. Compiler: Kode yang dikompilasi berjalan lebih cepat daripada Interpreter.
Interpreter: Kode yang ditafsirkan berjalan lebih lambat dari Compiler.
3. Compiler: Model Linking-Loading adalah model kerja dasar dari Compiler.
Interpreter: Model Interpretasi adalah model kerja dasar Interpreter.
4. Compiler: Kompiler menghasilkan output dalam bentuk (.exe).
Interpreter: Interpreter tidak menghasilkan output apa pun.
5. Compiler: Setiap perubahan dalam program sumber setelah kompilasi memerlukan kompilasi ulang seluruh kode.
Interpreter: Setiap perubahan dalam program sumber selama terjemahan tidak memerlukan terjemahan ulang seluruh kode.
6. Compiler: Kesalahan ditampilkan di Compiler setelah Compiling bersama-sama pada waktu saat ini
Interpreter: Kesalahan ditampilkan di setiap baris.
